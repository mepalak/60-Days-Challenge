# Day 20 – Face Puzzle Game with Camera 📸🧩

## 📌 Overview
On Day 20 of my **#60DayClaudeChallenge**, I explored AI-assisted front-end development by creating a **Face Puzzle Game**. Using a detailed prompt, Claude generated a complete browser-based puzzle game that captures a user's photo through the webcam, converts it into puzzle pieces, and provides an interactive drag-and-drop gameplay experience.

This project demonstrates how AI can accelerate the development of feature-rich web applications from a single prompt.

---

## 🎯 Objective
- Build a fully functional browser-based puzzle game.
- Capture images using the device webcam.
- Generate puzzle pieces from the captured photo.
- Implement drag-and-drop interactions.
- Track player performance with timers, moves, and leaderboard.

---

## 🛠️ Technologies Used
- HTML5
- CSS3
- JavaScript (Vanilla)
- HTML5 Canvas API
- MediaDevices API (`getUserMedia`)
- Local Storage API

---

## ✨ Features
- 📸 Live webcam access
- 🖼️ Capture user's face as puzzle image
- 🎮 Multiple difficulty levels (3×3, 4×4, 5×5)
- 🧩 Automatically generated puzzle pieces
- 🖱️ Mouse drag-and-drop support
- 📱 Touch gesture support for mobile devices
- ⏱️ Live timer with milliseconds
- 🔢 Move counter
- ✅ Correct piece highlighting
- 🏆 Automatic win detection
- 📊 Local leaderboard using Local Storage
- 🔄 Retake photo and replay options
- 📱 Fully responsive design

---

## 📂 Project Workflow
1. Request webcam permission.
2. Display live camera preview.
3. Capture user's face.
4. Select puzzle difficulty.
5. Slice the captured image into puzzle pieces.
6. Shuffle the pieces into a solvable arrangement.
7. Drag and swap pieces until the puzzle is completed.
8. Display final results and save the score locally.

---

## 📚 Learning Outcomes
- Working with the MediaDevices API
- Using Canvas for image processing
- Building interactive drag-and-drop interfaces
- Supporting both mouse and touch events
- Managing game state with JavaScript
- Using Local Storage for persistent data
- Creating responsive browser games

---

## 🚀 Key Takeaways
- AI can rapidly generate complex front-end applications from detailed prompts.
- HTML5 APIs make it possible to build engaging browser games without external frameworks.
- Proper prompt engineering significantly improves the quality of generated code.
- Combining Canvas, Camera APIs, and JavaScript enables interactive user experiences.
